## 1. Blog Canonical Tags ŌĆö Self-Canonicalisation

**Issue:** Every individual blog post was inheriting the canonical tag `https://jschoicegroup.com.au/blog` from `blog/layout.tsx`. Google treated all posts as duplicates of the listing page, blocking indexing.

**Files changed:**
- `src/app/(website)/blog/layout.tsx` ŌĆö Removed `alternates.canonical` entirely (layout-level canonicals are unsafe for dynamic routes).
- `src/app/(website)/blog/[slug]/page.tsx` ŌĆö Added self-canonical `https://jschoicegroup.com.au/blog/{slug}` to `generateMetadata`, plus full Open Graph article metadata (title, description, URL, `publishedTime`, author, featured image) and Twitter card.
- `src/app/(website)/blog/page.tsx` ŌĆö Changed relative canonical `/blog` to absolute `https://jschoicegroup.com.au/blog` for consistency.

## 5. Image `sizes` Prop ŌĆö Next.js Over-Sized Images

**Issue:** Three images were served at `w=3840` (4K) because `<Image fill>` was used without a `sizes` prop. Without `sizes`, Next.js assumes the image could fill any screen width and generates the largest possible URL, resulting in files well over 100 KB being served to all users.

**Files changed:**

| File | Image | `sizes` added |
|------|-------|---------------|
| `src/components/sections/home/About.tsx` | `about-2.webp` | `"(max-width: 1024px) 100vw, 50vw"` |
| `src/components/sections/about/FounderProfile.tsx` | `JanImage.jpeg` | `"170px"` (fixed-width container) |
| `src/components/sections/about/WhoWeAre.tsx` | `JanImage.jpeg` | `"(max-width: 1024px) 100vw, 45vw"` |
| `src/app/(website)/employment-education/page.tsx` | Unsplash photo | `"(max-width: 1024px) 100vw, 50vw"` |

## 9. Missing JSON-LD Structured Data ŌĆö Site-Wide

**Issue:** The site had zero JSON-LD structured data on 50+ pages. Only the root `layout.tsx` had a global `LocalBusiness` schema, and one landing page had a schema. This meant Google could not generate any rich results (FAQ snippets, breadcrumb trails, sitelinks) for any page on the site.

**Fix:**

Created a reusable component:

- `src/components/schema/JsonLd.tsx` ŌĆö renders `<script type="application/ld+json">` inline, works in both server and client components.

Added page-specific JSON-LD schemas to **36 pages** across the site.

**Schema mapping:**

| Page | Schemas Added |
|------|--------------|
| Home (`/`) | `WebSite`, `WebPage` |
| About (`/about-us`) | `AboutPage`, `Person` (Jan Fardowsi ŌĆō Founder) |
| Contact (`/contact-us`) | `ContactPage` |
| Blog listing (`/blog`) | `Blog` |
| Blog posts (`/blog/[slug]`) | `BlogPosting`, `BreadcrumbList` |
| 16 location pages (`/ndis-providers-*`) | `LocalBusiness`, `BreadcrumbList`, `FAQPage` |
| 4 service pages with FAQs | `Service`, `BreadcrumbList`, `FAQPage` |
| 11 service pages without FAQs | `Service`, `BreadcrumbList` |

**Location pages covered:**

Altona, Altona Meadows, Altona North, Craigieburn, Epping, Footscray, Hoppers Crossing, Lara, Laverton, Point Cook, Shepparton, South Morang, Tarneit, Truganina, Werribee, Williams Landing

**Service pages with FAQPage schema:**

Support Coordination, Transportation Assistance, Psychosocial Recovery Coaching, NDIS Accommodation Geelong

**Service pages with Service + BreadcrumbList only:**

Allied Health Services, Assistance with Daily Life, Assistance with Nursing Care, Access to Community Activities, Emergency Respite, Employment & Education, Group & Centre Activities, Innovative Community Participation, NDIS Access Requests, NDIS Accommodation, Client and Family Advocacy

**Implementation notes:**

- All `LocalBusiness` schemas on location pages reference `parentOrganization: { "@id": "https://jschoicegroup.com.au/#organization" }` ŌĆö linking back to the root layout's global schema.
- All `FAQPage` schemas derive directly from each page's own `const faqs` array. No hardcoding ŌĆö updating the FAQ array on a page automatically updates its structured data.
- Blog post `BlogPosting` schemas are fully dynamic, built from the Supabase post data (title, author, published date, featured image).
- For location pages and service pages that are `"use client"` components, schemas are defined as module-level constants before the component function and injected via `<JsonLd>` as the first child of `<main>`.
